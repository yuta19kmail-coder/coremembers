## 0. これは何か
小林モータース（自動車整備＋中古車・スタッフ約15名）の **正式な社員名簿＋組織＋スキル＋場所マスター** アプリ。
CoreFlow エコシステムの一員。単一HTMLファイル（インラインCSS/JS）で、house パターンに沿う。

- 本番URL：`https://coremembers.kobayashi-motors.com`
- ファイル：`D:\アプリ開発\CoreMembers\index.html`
- 最終目標（将来 Phase4-5）：AIによる横断的な指導履歴／人物Memoryの集約。**まだそこには行かない。今は土台（名簿・組織・権限・場所・スキル）を固める段階。**

### CoreMembers と portalMembers の関係（重要）
- **CoreMembers＝親**：全スタッフ（ログインしない週2バイトも含む）の人事マスター。
- **portalMembers＝子（アクセス層）**：CoreFlowにログインできる人だけ。`portalMemberId` で紐づけ。
- 「CoreFlowにアクセスできる＝全スタッフではない」。週2バイトは CoreMembers にだけ存在（portalMembers には作らない）。

## 1. インフラ / 接続
- Firebase（compat SDK 10.13.2）を **carflow-9d500 プロジェクトに相乗り**。
- Auth：Google ログイン（signInWithPopup）→ portalMembers で入室判定（落ちたら拒否画面）。
- authDomain は専用サブドメイン `coremembers.kobayashi-motors.com`（iOS Safari のログインループ回避）。
- Hosting site：`coremembers-km`（`.firebaserc` の target=coremembers）。
- Firestore コレクション（すべて `companies/{COMPANY_ID}/...` 配下）：
  - `coreMembers` … 社員
  - `coreDepts` … 部署（組織）
  - `coreLocations` … 場所マスター
  - `coreSettings/app` … アプリ設定（場所カテゴリ等）← **v0.4.0 で新設**
  - `userPrefs/{uid}` … uid↔memberId 橋渡し
  - `portalMembers`（読み取り）… ログイン/権限層

### 権限モデル（4段階）
`CM_ROLE` = オーナー / 管理者 / 責任者 / 一般。
- `IS_ADMIN = (オーナー || 管理者)` のみ編集可。
- オーナー＝portalMembers の master/admin。それ以外は portalMembers の coremembers.role を見る。

## 3. データモデル（主要フィールド）

### coreMembers（社員）
- 基本：`name, kana, ini, roleTitle(現場役職), primaryDeptId, subDeptIds[](兼務), managerId(上長)`
- **`topRole`（上位役割）** … 代表/副代表/専務/常務/取締役/顧問。部より上を“箱”でなく人の役割で表現（v0.5.0）。
- 雇用：`employmentType, status, joinedAt, weeklyDays, showInSchedule, schedulable`
- 連絡先：`ext, mobile, emailWork`
- 運転：`licenses[], mt, driveCaps[](左ハンドル/大型①/大型②), nearOnly(近場のみ), drivingConfidence, drivingNotes`
- 登録業務：`regTasks[]`（中古新規/名義変更/ナンバー変更/住所変更）
- **能力マトリクス：`locSkills = { [locId]: { status:'ok'|'never', tasks:[業務名,...] } }`**
  - status：✅行ける / 🔶行ったことない。tasks：その場所でできる業務（場所側 tasks に✅）。
  - 保存時 `goableLocationIds`（status==='ok'）を派生保存＝MHS/PitFlow互換。
- 生活圏：`homeArea`、スキル：`certs[], skillTags[], strengths, weaknesses`
- 連携：`portalMemberId`、アバター：`photo`（紐づけ時のスナップショット）
- ※ `color`（アイコン色）は **v0.4.1 で廃止**（アバター自動生成に移行）。`drivableSizes` も廃止。

### coreLocations（場所マスター）
- `name, aliases[](通称・AI照合用), category(キー), frequent(⭐よく使う), isBase(🏠会社/基準)`
- 座標：`lat, lng`（**手入力 or Googleマップ座標/URL貼り付けで取得。自動ジオコーディングは廃止**）
- `address(表示・地図リンク用), memo, active, order`
- **`tasks[]`（その場所で発生する業務）** … ＋追加式。一覧では組織図のように枝でぶら下げ表示。
- ※ `area`（手入力方面）は廃止。方面は基準地点(isBase)からの方角・距離を座標から自動算出して表示。

### coreDepts（部署）
- `name, type(部/課/班/拠点), parentId(上位部署), leaderMemberId(責任者), color, order`

### coreSettings/app
- `locCats: [{key,label}, ...]` … 場所カテゴリ（設定ページで編集・並び替え・追加・削除）。
  - 未設定時は `DEFAULT_LOC_CATS`（陸運局/部品商/取引先/自社拠点/顧客先/**警察署**/その他）にフォールバック。

## 4. 画面 / 機能（現状）
- **社員名簿**：部署ごとにグルーピング、アバター付きカード、検索、在籍フィルタ、portalMembersから取り込み。
- **組織（部署）**：最上部に「上位役割」ストリップ（代表・役員などを役割バッジで）→ その下に部署ツリー。
- **場所マスター**：並び順＝①自社/基準 → ②⭐よく使う（カテゴリ→五十音）→ ③カテゴリ→五十音。業務を枝でぶら下げ表示。
- **⚙️設定**（v0.4.0新設）：場所カテゴリの名称変更・並び替え・追加・削除。管理者のみ。
- **About**：接続状況・ログアウト。

### アバター仕様（v0.4.1）
- portalMembers 紐づけ済み → **その人の“今の”Googleアバターを引く**（`memberPhoto()` が portalById で最新を参照。アプリ再読込で追従）。
- 紐づけなし（手動追加） → **苗字頭文字＋名前ハッシュ由来の色**でGoogle風アバターを自動生成（`avatarColor()` / `avatarHTML()`）。
